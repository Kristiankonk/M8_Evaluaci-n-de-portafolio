# Informe de Portafolio Profesional  
## Cristian Ramírez – Desarrollador Full Stack

### 1. Introducción

El presente documento tiene como objetivo presentar el desarrollo y la implementación de un portafolio profesional orientado al área de desarrollo de software. En él se integran dos productos tecnológicos que responden a problemáticas reales: una Tienda de Coleccionables y un Sistema de Reservas. 

A través de estos proyectos se busca evidenciar el dominio de herramientas, metodologías y buenas prácticas en el desarrollo de aplicaciones web, así como la capacidad para trasladar requerimientos funcionales a soluciones concretas.

---

### 2. Descripción del Portafolio

El portafolio profesional se encuentra estructurado como un sitio web responsivo que incluye:

- Una sección de presentación personal.
- Una sección de proyectos destacados.
- Enlaces a repositorios de GitHub y futuras demostraciones en línea.
- Información de contacto para empleadores o colaboradores.

El diseño se realizó siguiendo criterios de claridad, organización, estética profesional y enfoque en la navegabilidad.

---

### 3. Proyecto 1: Tienda de Coleccionables

#### 3.1 Problema abordado

La gestión manual de productos coleccionables, pedidos y clientes puede generar errores, pérdida de información e ineficiencias en el seguimiento de ventas. El proyecto propone una aplicación web que centraliza estos procesos.

#### 3.2 Tecnologías utilizadas

- Python, Django.
- MySQL.
- HTML, CSS, Bootstrap.
- Git y GitHub para control de versiones.

#### 3.3 Enfoque de desarrollo

El desarrollo siguió las etapas de:

1. Levantamiento de requerimientos.
2. Diseño del modelo de datos (productos, categorías, clientes, pedidos).
3. Implementación de vistas, formularios y plantillas.
4. Integración de un carrito de compras basado en sesión.
5. Validación de stock y creación de pedidos.
6. Pruebas funcionales básicas sobre los principales flujos (agregar productos, comprar, actualizar stock).

---

### 4. Proyecto 2: Sistema de Reservas

#### 4.1 Problema abordado

Muchos servicios (consultorías, atenciones, clases) requieren agendar horas. Cuando este proceso se realiza de forma manual, es frecuente la sobreasignación de horarios, la falta de historial de reservas y la poca claridad de la disponibilidad.

#### 4.2 Tecnologías previstas

- Django o framework equivalente.
- Base de datos relacional (MySQL/PostgreSQL).
- HTML, CSS, Bootstrap.
- Git y GitHub.

#### 4.3 Enfoque de desarrollo

El enfoque contempla:

1. Definición de entidades (Usuario, Servicio, Reserva).
2. Implementación de un flujo de autenticación.
3. Creación de formularios para administrar servicios y reservas.
4. Validación de disponibilidad al momento de crear una reserva.
5. Diseño de interfaces simples y claras para visualizar la agenda.

---

### 5. Gestión del Código y Portafolio

Ambos proyectos se gestionan utilizando Git y GitHub, con repositorios independientes, README documentados y una estructura de carpetas coherente. El portafolio integra los enlaces a dichos repositorios, lo que permite a cualquier evaluador revisar el código fuente y la organización del proyecto.

---

### 6. Conclusiones

El desarrollo del portafolio y de los proyectos asociados ha permitido consolidar competencias técnicas en desarrollo web, modelado de datos y uso de frameworks modernos. Asimismo, ha contribuido a fortalecer habilidades transversales como la planificación, la documentación y la comunicación de soluciones tecnológicas.

Este portafolio se proyecta como una herramienta central para la búsqueda laboral, al ofrecer evidencia concreta del trabajo realizado y del potencial de crecimiento en el ámbito del desarrollo de software.
